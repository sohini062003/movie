Flow Of Work

I. Training an ML Model for Movie Similarity


(Hover)

The ML model was trained using movie metadata, such as title, overview, and keywords, which were preprocessed using NLP techniques (tokenization and stop-word removal). TF-IDF vectorization was used to convert text into numerical vectors, capturing the importance of terms in each movie description. Cosine similarity was employed to calculate the angular distance between vectors, allowing the system to find movies most similar to a given query. The model was fine-tuned and tested using TMDB datasets to ensure accurate and relevant recommendations.

II. Building Backend with Flask & TMDB API Integration


(Hover)

Flask was used as the backend framework to connect the ML model with the TMDB API. When users search for a movie, Flask processes the query, retrieves similar movies using the trained model, and fetches additional metadata (e.g., titles, posters) from the TMDB API. Flask routes handle communication between the frontend and backend, ensuring efficient data delivery and smooth performance.

III. Implementing Genre-Based Recommendations


(Hover)

Movie genres were retrieved from the TMDB API and mapped to each movie in the database. Users can select a genre (e.g., Action, Comedy) from the interface, and Flask fetches popular or top-rated movies from that genre via the TMDB API. This filter-based approach allows personalized exploration of movies based on user preferences.

IV. Frontend Development for Recommendations


(Hover)

The frontend was designed with HTML, CSS, and JavaScript to ensure an intuitive and visually appealing user experience. Recommendations are displayed as clickable movie cards, with posters and titles arranged in a user-friendly layout. Interactive features like search bars, buttons, and carousels enhance user engagement, while JavaScript handles dynamic content updates for seamless interactivity.

V. Fetching and Displaying Detailed Movie Information


(Hover)

Additional movie details (e.g., cast, crew, runtime, trailers, streaming platforms) were fetched using the TMDB API. A "More Info" button was added to every recommended movie, allowing users to view detailed information. The frontend was styled with CSS to display this data in a clean, organized, and responsive format, improving decision-making for users.

VI. Sentiment Analysis for Movie Reviews


(Hover)

Model Training: The sentiment analysis model was trained using the IMDB dataset from Kaggle. Text preprocessing involved cleaning, tokenization, and vectorization with Bag-of-Words. Bernoulli Naive Bayes achieved over 85% accuracy and was saved using pickle for integration.
Flask API Integration: Flask fetched movie reviews from the TMDB API, processed them with the sentiment analysis model, and categorized them as positive or negative.
Frontend Development: A "Reviews" page was built using HTML, CSS, and JavaScript to display reviews. Positive reviews were highlighted in green, negative ones in red, and a sentiment summary was provided for better user experience.

VII. Deploying the application


(Hover)

Once the app was fully developed, it was deployed using Render, a reliable platform for hosting web applications. The project code was first pushed to GitHub, ensuring version control and easy collaboration. Render was then configured to pull the code directly from the GitHub repository, allowing seamless deployment. With just a few clicks, the app was live and accessible to users, providing an intuitive interface for movie recommendations. This step ensured that people could access the app anytime, anywhere, and enjoy its features with ease.

1. IMDB Dataset for Sentiment Analysis

Source:

Kaggle IMDB Dataset

Description:

This dataset consists of 50,000 labeled movie reviews from IMDB, evenly split between positive and negative sentiments. It is widely used for text classification and natural language processing tasks.

Key Features:

- review: The text of the movie review.
- sentiment: Label indicating whether the review is positive or negative.

2. TMDB 5000 Movie Dataset by TMDB | from Kaggle

This dataset provides comprehensive metadata for 5000 movies, ideal for building recommendation systems or analyzing trends in the movie industry. It includes two files:

tmdb_5000_credits.csv :

Contains detailed information about the cast, crew, and roles associated with each movie. This data is useful for analyzing the involvement of actors and directors in various projects.

tmdb_5000_movies.csv :

Includes essential metadata such as movie titles, genres, budgets, revenues, popularity, and vote statistics. Perfect for understanding box office performance, genre popularity, and audience engagement over time.
